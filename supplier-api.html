    Overview

    Welcome to the DerbySoft Supplier API documentation! You can implement this API to let DerbySoft sync ARI from you. DerbySoft uses booking API to deliver distributor's booking to you. Also, you can call our API to trigger DerbySoft to do a full refresh.

    You can view examples in the dark area to the right.

    HTTPS for all request. Method is POST.

    Authentication

    DerbySoft supports Access Token authentication method. You should generate an access token for DerbySoft to access your endpoint. Here is a example of our HTTP request header.

    POST /api/v0.1/availability HTTP/1.1
Authorization: 53ac07777cdffac2d53000002d698728ce964432d7167596bc005c5fc

    Language & Charset

    • English is the only language you need to support now
    • All message is under charset UTF-8
    • All description fields like comments, first name, last name, cancellation description and fee name are all English characters
    • DerbySoft will do romanization if distributor sends non-latin characters on reservation request.

    Common Fields

    Code Format

    • Channel Code - [0-9] [A-Z] and '-'
    • Hotel Code - [0-9] [A-Z] and '-'
    • Room Type - [0-9] [A-Z] and '-'
    • Rate Plan - [0-9] [A-Z] and '-'
    • IATA - [0-9]
    • ChannelResID - DerbySoft can not control it. But most of channel reservation code is [0-9] [a-Z] and [- _]
    • DerbyResID - [0-9] [A-Z]
    • HotelResID - [0-9] [A-Z]

    Header

    The header is included in all request and response.

    Parameter Count Type Size Description
    Header 1 Header of request and response
    ....Version 1 String 20 Version of the API
    ....EchoToken 1 String 64 Unique id of request and response
    ....Channel 1 String 32 Channel unique code
    ....IATA 0.1 String 20 IATA number for the request. Some channels send different IATA for booking request.
    ....Language 0.1 String 20 Language code. Default is en.

    Booking API

    DerbySoft uses booking API to do live check and booking delivering. There are two booking models:

    1. Booking Request Model (ForceSell = false)
      • Booking confirmed by the hotel. Distributor accepts hotel rates.
    2. Booking Notification Model ( ForceSell = true) - Will support in the future
      • Booking confirmed by the distributor. Hotel accepts distributor rates.

    ARI API

    1. Support Change API
      • DerbySoft call Change API every 5-15 minutes to know if hotel has changes
      • Call Rate, Avail Status, Inventory API to get data back and save to database
    2. Not Support Change API
      • DerbySoft call Rate, Avail Status, Inventory API timely to get data back and save to database
      • DerbySoft can setup different interval for ARI
    DerbySoft recommends you to support GZIP for reducing the message size.
    You can call `TriggerRefresh` API to trigger DerbySoft to do a full refresh. It isn't ready now. We will provide the API detail later.

    Appendix

    Cancellation Policy

    Cancellation policy defines what penalty will be charge when a guest cancels the booking at certain advance time range. The penalty is related to no show or a time range before no show.

    Time Range

    • No Show. The customer doesn't cancel the booking or cancel it after arrival day. Most of hotel define the no show time is 12 AM -10 AM of next day of arrival. The last penalty without advance day is the penalty of no show.
    • Deadline on arrival day- No Show. Many hotels have a deadline on arrival day and will apply different penalty when customer cancel the booking after the deadline. The format of deadline is 4 PM or 6:30 PM. The default deadline is 12 AM.
    • Advance day. The format is 1D, 2D, 7D or AD. 1D means 24 hours before 'Deadline'. AD means unlimited advance day.
    • Timezone. The timezone of the deadline or advance day is hotel local timezone.

    Penalty Type

    There are three penalty types.

    • Percentage. Will charge the percentage of the whole stay. For example, 80P, 100P
    • Amount. Will charge fix amount. The currency of this amount is same as rate. For example, 120, 245.75
    • Nights. Will charge the fee of nights. For example, 1N, 2N.

    Examples

    The checkin day of below example is 2016-10-15.

    A Full Example

    AD20_7D50_3D1N_1D50P_6PM0D80P_100P

    • 100P is the penalty of no show. It means 100 percent of the whole stay will be charged if customer doesn`t cancel the booking and not arrive at hotel.
    • 6PM0D80P defines the penalty between 6 PM of arrival day(0D) and no show. It means 80 percent of whole stay will be charge if you cancel the booking between 6 PM and 11:59 PM of 2016-10-15.
    • 1D50P defines the penalty between 2016-10-14 6 PM and 2016-10-15 6 PM. It means 50 percent of the whole stay will be charged if you cancel the booking between 2016-10-14 6 PM and 2016-10-15 6 PM.
    • 3D1N defines the penalty between 2016-10-12 6 PM and 2016-10-14 6 PM. It means first night will be charged if you cancel the booking between 2016-10-12 6 PM and 2016-10-14 6 PM.
    • 7D50 defines the penalty between 2016-10-08 6 PM and 2015-10-12 6 PM. It means 50$ will be charged if you cancel the booking between 2016-10-18 6 PM and 2016-10-12 6 PM.
    • AD20 defines the penalty between booking time and 2015-10-12 6 PM. It means 20$ will be charged if you cancel the booking between the time you make the reservation and 2016-10-12 6 PM.

    Fee

    The Name of one fee need to be consist. For same tax, you MUST always give us the same Name. The charge type of Fee should be fixed amount or percentage. They both should not present in one Fee. So there is Amount with Unit or Percent for one Fee node.

    Inclusive & Exclusive

    • Inclusive - It means the fee is included in AmountBeforeTax.
    • Exclusive - It means the fee isn't included in AmountBeforeTax.

    Charge Type - Amount

    The currency of amount is same as rate. There are 4 units of fixed amount tax. Unit is required if Amount isn't null.

    Per Night

    • PerRoomPerNight - The Amount will be charge every room and every night.
    • PerPersonPerNight - The Amount will be charge every adult in the room and every night.

    Per Stay

    • PerRoomPerStay - The Amount will be charge every room for the whole stay.
    • PerPersonPerStay - The Amount will be charge every adult in the room for the whole stay.
    For some distributors only need `AmountAfterTax` like HotelBeds, they manage per stay tax by themselves to reduce ARI complexity. DerbySoft only applies non-PerStay tax rules to calculate `AmountAfterTax`.

    Charge Type - Percent

    Percent is quite simple. It's the percentage of the rate on each day.

    Cumulative Tax

    We don't support Cumulative Tax in the spec. You can use below solution to solve cumulative tax calculation.

    • PerRoomPerNight and PerPersonPerNight cumulative tax - You can put it to AmountBeforeTax and mark this tax to Inclusive.
    • Percent cumulative tax - You can add it to other Percent tax. For example, if you have a 6% cumulative tax and a 10% normal percentage tax. You can merge them to 16.6% percent tax.

    Tax Change

    One tax could change by government. For example,

    • CheckIn: 28 Dec
    • CheckOut: 3 Jan
    • Tax1: City Tax 2$ PerRoomPerNight
    • Tax2: State Tax 8.5%
    • Daily Rate: 100$

    The state tax changes from 8.5% to 9.5% next year. There are 2 possible calculation logic of above stay:

    1. Apply CheckIn Tax - AmountAftaxTax of each day = 100$ * (1+8.5%) + 2$ = 110.5$. Total After Tax = 110.5$ * 6 = 663.0$.
    2. Apply Daily Tax - AmountAfterTax of day before 1 Jan = 100$ * (1+8.5%) + 2$ = 110.5$. AmountAftaxTax of day after 31 Dec = 100$ * (1+9.5%) + 2$ = 111.5$. Total Aftre Tax = 110.5$ * 4 + 111.5 * 2 = 665.0$

    Please let DerbySoft know your logic then we can config it.

    FAQ

    Mapping

    • There is no mapping between DerbySoft and you.
    • DerbySoft will do necessary mappings between DerbySoft and distributor.

    Rate Plan Sharing

    DerbySoft supports rate plans sharing. DerbySoft uses All as channel code on Change and Availability API to get all rates back. You need to provide rate plan and room type spreadsheet for each distributor to DerbySoft so we can config it on our system. We will provide API to you to update this information in the future.

    AmountBeforeTax, AmountAfterTax & Fees

    The AmountAfterTax is the final amount that guest or distributor needs to pay. For example, you charge it from the guest credit card or VCC. AmountAfterTax could not include the fee that hotel have to charge the guest directly when the guest at the hotel.

    • The best is you return AmountBeforeTax, AmountAfterTax and Fees.
    • It's Ok if you only can return AmountAfterTax.
    • Please return both AmountBeforeTax and Fees because some distributors need AmountAfterTax.

    Child Pricing

    • It's possible that distributor pass child count without child ages on Availability and Reservation request. You need to setup a default child age like 12. This default age is based on the business term between you and distributors.

    Loading & Static Content

    • We are planning to add static content API then DerbySoft can sync content from you by the API.
    • You can download a standard content spreadsheet on our website. Please send the spreadsheet with new/updated hotel information back to our support email. DerbySoft support team will load them and notice you once done.

    Accuracy Reporting

    DerbySoft will generate an accuracy report every day based on you availability API. We randomly call you availability API and compare to our cache to see the rates are same or not.

    Monitoring

    DerbySoft provides 24X7 service to our customers. You can always call our 24X7 helpdesk for the urgent event. Also, our 24X7 team will call your 24X7 hotline to report the urgent event.
